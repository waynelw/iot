## 1. 文档目标

本文档用于指导物联网管理系统的：

- 上线前检查
- 发布执行
- 上线后观察
- 常见故障处理
- 应急回滚
- 事后复盘

适用对象：`DevOps/SRE`、`值班工程师`、`后端负责人`、`技术负责人`。

---

## 2. 上线前检查清单

## 2.1 版本与变更检查

- 发布版本已冻结
- 变更范围已评审
- 相关配置变更已确认
- 数据库变更脚本已评审
- 回滚方案已准备
- 发布窗口已确认

## 2.2 环境与依赖检查

- K8s 集群状态正常
- Kafka/PG/Redis/EMQX 状态正常
- 对象存储可访问
- IAM 登录可用
- 域名与证书可用
- 监控与告警可用

## 2.3 数据与安全检查

- 数据库已备份
- Secret 已更新并验证
- 敏感配置未误写入 ConfigMap
- 审计日志链路可用
- 权限模型已验证

## 3. 标准发布步骤

### 步骤 1：发布前确认

- 确认值班人员在线
- 确认变更单与发布单已通过
- 确认压测与演练报告已满足上线条件

### 步骤 2：发布配置与镜像

- 更新 Helm values
- 更新镜像版本
- 校验渲染结果
- 确认 ConfigMap/Secret 差异

### 步骤 3：灰度发布

- 先发布低风险服务
- 再发布核心业务服务
- 最后发布接入链路相关服务
- 每发布一批后观察关键指标 15~30 分钟

### 步骤 4：发布验证

- 验证健康检查
- 验证登录
- 验证设备接入
- 验证遥测上报
- 验证命令下发
- 验证后台查询

### 步骤 5：发布完成

- 记录发布时间、版本号、异常项
- 通知相关团队进入观察期

## 4. 上线后观察项

## 4.1 核心指标

## 5. 常见故障处理手册

## 5.1 故障：设备大量连接失败

### 现象

- MQTT 连接数下降
- 鉴权失败率上升
- 设备集中重试

### 排查步骤

- 检查 `mqtt-auth-service` 日志
- 检查 EMQX 鉴权 Hook 延迟
- 检查 Redis/PG 是否异常
- 检查 Secret 是否误更新
- 检查最近是否有 Topic 权限变更

### 处理动作

- 回滚最近鉴权配置变更
- 扩容 `mqtt-auth-service`
- 降低非关键日志级别，释放 CPU
- 必要时限流重连风暴

## 5.2 故障：Kafka lag 持续增长

### 现象

- 消息消费滞后持续升高
- 规则、告警、影子更新延迟明显

### 排查步骤

- 检查消费者组是否异常
- 检查 `message-ingest-service`、`rule-service`、`command-dispatcher` CPU/内存
- 检查 Kafka Broker 磁盘与网络
- 检查是否出现异常消息风暴

### 处理动作

- 临时扩容消费者副本
- 降低低优先级消费任务
- 增加 Topic 分区（如允许）
- 暂停非关键批量任务

## 5.3 故障：命令下发超时率上升

### 现象

- 命令状态停留在 `ACCEPTED` 或 `DISPATCHED`
- 用户反馈设备控制无响应

### 排查步骤

- 检查 `command-service`、`command-dispatcher` 日志
- 检查设备在线状态是否准确
- 检查 MQTT Topic 投递是否正常
- 检查 ACK 回执 Topic 是否积压

### 处理动作

- 扩容 `command-dispatcher`
- 核查 Redis 在线状态缓存
- 暂停批量命令任务
- 必要时切换为限速模式

## 5.4 故障：查询接口延迟升高

### 现象

- 管理后台卡顿
- 设备列表或详情加载超时

### 排查步骤

- 检查 `query-service` CPU/内存
- 检查 PostgreSQL 慢查询
- 检查 Redis 命中率
- 检查是否有大分页或复杂筛选请求

### 处理动作

- 限制大分页查询
- 临时增加只读实例或扩容 `query-service`
- 优化慢查询 SQL 与索引
- 启用热点缓存

## 5.5 故障：Redis 异常或切换

### 现象

- 在线状态不准确
- 影子查询波动
- 命令状态缓存异常

### 排查步骤

- 检查 Redis 主从状态
- 检查客户端超时与重试配置
- 检查网络抖动

### 处理动作

- 切换到从持久化库重建热点缓存
- 降级部分非关键缓存能力
- 调整 Redis 客户端连接池与超时配置

## 5.6 故障：PostgreSQL 主库压力过高

### 现象

- 接口整体变慢
- 连接数飙升
- 慢查询激增

### 排查步骤

- 检查慢查询日志
- 检查连接池配置
- 检查热点 SQL
- 检查批量任务是否在高峰运行

### 处理动作

- 停止非关键批量作业
- 将读流量切到只读库
- 临时提高连接池限制或限流请求
- 优化索引与 SQL

## 6. 回滚手册

### 触发条件

- 接入成功率显著下降
- 命令超时率超阈值
- 5xx 错误率超阈值
- Kafka lag 持续不可控
- 核心服务健康检查异常

### 回滚步骤

- 停止继续发布
- 执行 `helm rollback`
- 回滚 ConfigMap/Secret 变更
- 验证恢复后核心指标
- 记录回滚原因与时间点
- 通知业务与管理方

## 7. 值班与升级机制

| 等级 | 触发条件 | 响应人 |
|---|---|---|
| P1 | 接入主链路中断、命令大面积失败 | 值班 SRE + 后端负责人 + 技术负责人 |
| P2 | 查询性能明显下降、部分告警延迟 | 值班 SRE + 对应模块负责人 |
| P3 | 个别页面异常、局部功能故障 | 值班工程师 |

### 升级原则

- P1 事故 5 分钟内拉群响应
- P1 事故 15 分钟内给出初步影响范围
- P1 事故 30 分钟内给出处理策略
- 重大事故必须产出复盘报告
