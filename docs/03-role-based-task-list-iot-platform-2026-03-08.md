# 物联网管理系统按角色拆分的任务清单

**Document Version:** 1.0  
**Date:** 2026-03-08  
**Author:** System Architect  
**Status:** Draft

---

## 1. 文档目标

本清单基于 `20 万设备` 物联网管理系统建设方案，将实施任务按角色拆分为：

- 后端
- 前端
- 测试
- DevOps/SRE

目标是让项目经理、技术负责人和各小组负责人可以直接据此拆分迭代任务、分配责任人并跟踪执行状态。

---

## 2. 角色职责边界

| 角色 | 责任边界 |
|---|---|
| 后端 | Go 微服务开发、数据库设计、Kafka/Redis 集成、业务 API、设备链路 |
| 前端 | 管理后台、设备页、命令页、OTA 页、告警页、权限界面 |
| 测试 | 接口测试、联调测试、回归测试、性能测试、故障场景验证 |
| DevOps/SRE | K8s、中间件、CI/CD、监控告警、部署发布、容量治理 |

## 3. 后端任务清单

## 3.1 基础工程任务

- 初始化 Go 项目结构：`cmd/`、`internal/`、`pkg/`、`api/`
- 建立统一配置加载组件
- 建立统一日志组件
- 建立统一 Trace 组件
- 建立 PostgreSQL 客户端封装
- 建立 Redis 客户端封装
- 建立 Kafka Producer/Consumer 封装
- 建立统一错误码与错误处理机制
- 建立统一鉴权中间件与租户上下文中间件
- 建立 OpenAPI / gRPC 接口定义规范

## 3.2 设备接入与认证任务

- 实现 `mqtt-auth-service`
- 实现设备身份校验
- 实现设备 Topic 权限校验
- 实现设备密钥轮换接口
- 实现冻结/吊销设备拒绝接入逻辑
- 实现鉴权日志记录
- 输出设备接入异常错误码

## 3.3 设备管理任务

- 实现 `device-service`
- 实现产品模型管理
- 实现设备注册、激活、冻结、吊销
- 实现设备标签、设备分组管理
- 实现设备分页查询接口
- 实现设备详情接口
- 实现设备导入/导出接口
- 建立设备生命周期状态机

## 3.4 消息与影子任务

- 实现 `message-ingest-service`
- 实现上行消息格式校验
- 实现消息标准化转换
- 实现 Broker 到 Kafka 的消息桥接
- 实现 `shadow-service`
- 实现 Desired/Reported 模型
- 实现影子版本控制
- 实现在线状态缓存与最近活跃时间更新
- 实现影子查询接口

## 3.5 命令与控制任务

- 实现 `command-service`
- 实现 `command-dispatcher`
- 建立命令状态机
- 实现命令幂等键
- 实现命令超时与重试策略
- 实现设备回执处理逻辑
- 实现命令历史查询与状态追踪接口
- 实现批量命令限流控制

## 3.6 规则、告警与 OTA 任务

- 实现 `rule-service`
- 实现阈值、持续超限、离线等规则
- 实现 `alarm-service`
- 实现告警去重、抑制、恢复逻辑
- 实现通知编排（Webhook/短信/邮件）
- 实现 `ota-service`
- 实现固件元数据管理
- 实现 OTA 升级任务创建、暂停、取消、进度回传
- 实现灰度升级策略

## 3.7 查询与开放接口任务

- 实现 `query-service`
- 实现设备列表与设备详情聚合查询
- 实现命令记录查询
- 实现告警查询接口
- 实现 OTA 任务查询接口
- 实现 `openapi-service`
- 实现 `webhook-service`
- 实现接口限流与审计埋点

## 3.8 安全与审计任务

- 对接 IAM/OIDC
- 实现 RBAC
- 实现租户隔离
- 实现审计日志落库
- 实现敏感操作审计
- 实现接口权限校验
- 实现服务间调用身份校验

## 3.9 后端交付物

- Go 微服务代码
- 数据库建表脚本
- Kafka Topic 初始化脚本
- OpenAPI 文档
- 服务 README
- 单元测试与接口测试报告

## 8. 任务分配建议

- 每个角色按 `里程碑 -> 周计划 -> 日任务` 三层拆解
- 每周必须输出：已完成、进行中、阻塞项、风险项
- 关键链路必须指定唯一责任人，避免多人共管
- 前端、后端、测试至少提前一周冻结联调范围
- DevOps 必须先于功能开发完成基础环境与发布链路
